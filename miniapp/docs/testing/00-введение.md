# Документация для тестировщиков MAIN Community MiniApp

## 1. Цель документации

Данная документация предназначена для тестировщиков приложения MAIN Community MiniApp — Telegram мини-приложения для нетворкинга и управления событиями сообщества MAIN.

Документация содержит:
- Правила и стандарты тестирования
- Полное описание функций приложения
- Чеклисты для каждого экрана
- Детальные тест-кейсы для критичных функций
- Шаблоны отчетов и баг-репортов

---

## 2. Общие правила тестирования

### 2.1 Подготовка к тестированию

1. **Перед началом тестирования:**
   - Убедитесь, что у вас есть доступ ко всем тестовым аккаунтам
   - Проверьте версию Telegram (требуется 8.0+)
   - Ознакомьтесь с актуальной версией приложения
   - Прочитайте release notes (если есть)

2. **Во время тестирования:**
   - Фиксируйте все найденные баги сразу
   - Делайте скриншоты/видео проблем
   - Записывайте шаги воспроизведения
   - Проверяйте на разных устройствах (iOS/Android)

3. **После тестирования:**
   - Заполните отчет о тестировании
   - Убедитесь, что все баги зарегистрированы
   - Вернитесь к critical багам для ретеста

### 2.2 Приоритеты тестирования

| Приоритет | Описание | Примеры функций |
|-----------|----------|-----------------|
| **P1 (Critical)** | Блокирует основной функционал | Авторизация, регистрация на событие, check-in |
| **P2 (High)** | Важный функционал с обходным путем | Редактирование профиля, свайпы, XP система |
| **P3 (Medium)** | Второстепенный функционал | Настройки, социальные ссылки, скины |
| **P4 (Low)** | Косметические проблемы | UI несоответствия, мелкие улучшения |

### 2.3 Классификация багов по серьезности

| Серьезность | Описание | Пример |
|-------------|----------|--------|
| **Critical** | Приложение падает, данные теряются | Crash при открытии, потеря регистрации |
| **Major** | Функция не работает | Невозможно зарегистрироваться на событие |
| **Minor** | Функция работает с ограничениями | Неправильное форматирование даты |
| **Trivial** | Косметический дефект | Опечатка в тексте, неровные отступы |

## 3. Типы тестирования

### 3.1 Smoke-тестирование
**Цель:** Быстрая проверка работоспособности основных функций после деплоя.

**Чеклист Smoke-теста:**
- [ ] Приложение открывается в Telegram
- [ ] Авторизация проходит успешно
- [ ] Главная страница загружается
- [ ] Можно перейти на все вкладки
- [ ] Профиль отображается корректно
- [ ] Список событий загружается

### 3.2 Регрессионное тестирование
**Цель:** Проверка, что новые изменения не сломали существующий функционал.

**Когда проводить:**
- После каждого релиза
- После значительных изменений в коде
- Перед важными событиями сообщества

### 3.3 Acceptance-тестирование
**Цель:** Проверка соответствия требованиям заказчика.

**Проводится:**
- Перед финальным релизом новой функции
- По чеклистам от product owner

## 4. Критичные функции (обязательное тестирование)

Следующие функции **ВСЕГДА** должны быть протестированы перед релизом:

1. **Авторизация через Telegram**
   - Получение данных пользователя
   - Создание/обновление профиля
   - Валидация init_data

2. **Регистрация на событие**
   - Просмотр списка событий
   - Регистрация с получением билета
   - Отмена регистрации
   - Начисление +10 XP

3. **Check-in на событии**
   - Сканирование QR-кода волонтером
   - Подтверждение посещения
   - Начисление +50 XP

4. **Система свайпов и матчей**
   - Просмотр профилей
   - Like/Skip/Superlike
   - Создание матча при взаимном лайке
   - Уведомление о матче

5. **Редактирование профиля**
   - Все поля профиля
   - Валидация данных
   - Сохранение изменений

6. **Система XP и рангов**
   - Начисление XP за действия
   - Повышение ранга
   - Отображение прогресса
