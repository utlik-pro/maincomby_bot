# Классификация дефектов

## По серьезности (Severity)

| Уровень | Описание | Примеры |
|---------|----------|---------|
| **Critical** | Приложение падает, данные теряются, безопасность под угрозой | Crash при открытии, потеря регистрации на событие, утечка данных |
| **Major** | Основная функция не работает, нет обходного пути | Невозможно зарегистрироваться, свайпы не работают, XP не начисляется |
| **Minor** | Функция работает с ограничениями или есть обходной путь | Неправильный формат даты, мелкие UI баги, медленная загрузка |
| **Trivial** | Косметический дефект, не влияет на функциональность | Опечатка, неровные отступы, несоответствие дизайну |

## По приоритету (Priority)

| Уровень | Описание | Срок исправления |
|---------|----------|------------------|
| **P1** | Критический, блокирует релиз | Немедленно |
| **P2** | Высокий, должен быть исправлен в этом релизе | До релиза |
| **P3** | Средний, может быть исправлен в следующем релизе | Планируется |
| **P4** | Низкий, исправить когда будет время | Backlog |

# Примеры баг-репортов

## Пример 1: Critical баг

| Поле | Значение |
|------|----------|
| **ID** | BUG-001 |
| **Заголовок** | Приложение падает при открытии профиля после обновления |
| **Серьезность** | [x] Critical |
| **Приоритет** | [x] P1 |

### Шаги воспроизведения
1. Открыть приложение через Telegram
2. Перейти на вкладку "Профиль"
3. Приложение закрывается с ошибкой

### Ожидаемый результат
Открывается экран профиля с данными пользователя

### Фактический результат
Приложение падает, возвращает на главный экран Telegram

### Воспроизводимость
[x] Всегда (100%)

---

## Пример 2: Major баг

| Поле | Значение |
|------|----------|
| **ID** | BUG-002 |
| **Заголовок** | XP не начисляется после check-in на событии |
| **Серьезность** | [x] Major |
| **Приоритет** | [x] P2 |

### Шаги воспроизведения
1. Войти под тестовым аккаунтом @test_free_user
2. Зарегистрироваться на тестовое событие "Test Workshop"
3. Войти под волонтером @test_volunteer
4. Отсканировать QR-код билета
5. Проверить XP пользователя @test_free_user

### Ожидаемый результат
XP пользователя увеличивается на 50

### Фактический результат
XP остается прежним, check-in отмечен как успешный

### Воспроизводимость
[x] Всегда (100%)

### Дополнительная информация
- В базе данных статус регистрации = 'attended'
- Запись в xp_transactions отсутствует

---
